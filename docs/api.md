# Rover API Reference

The Rover backend API is built using FastAPI. It processes GitHub App callbacks, validates webhooks, triggers codebase scans, and initiates fixing agents.

## Base URL
* Local development: `http://localhost:8000`
* Production: `https://your-api.onrender.com`

---

## 🛣️ API Endpoints

### 1. `GET /`
Standard liveness probe to verify that the API service is active.

* **Response Status**: `200 OK`
* **Response Body**:
  ```json
  {
    "status": "Rover is running",
    "version": "1.0"
  }
  ```

---

### 2. `GET /github/callback`
Processes the authorization redirect from the GitHub App installation flow. Saves the `installation_id` and redirects the user back to the Streamlit Dashboard.

* **Query Parameters**:
  - `installation_id` (Integer, Required): The installation ID generated by GitHub.
  - `setup_action` (String, Optional): The setup context string.
* **Response Status**: `307 Temporary Redirect` (redirects to `DASHBOARD_URL`).
* **Error Responses**:
  - `400 Bad Request`: If `installation_id` is missing.

---

### 3. `POST /scan`
Triggers an asynchronous scan of a repository. Clones the repository, runs AST rules, applies LLM checks, and ranks findings.

* **Request Body** (JSON):
  ```json
  {
    "repository_url": "https://github.com/owner/repo",
    "async": true
  }
  ```
* **Response Status**: `200 OK` or `202 Accepted`
* **Response Body**:
  ```json
  {
    "scan_id": "scan-1784201895829",
    "repository": "https://github.com/owner/repo",
    "status": "scanning",
    "bugs": [],
    "files_scanned": 0,
    "ignored_files": 0,
    "scan_duration_seconds": 0
  }
  ```
* **Error Responses**:
  - `400 Bad Request`: Invalid GitHub URL or missing installation ID (if using App authentication).
  - `403 Forbidden`: GitHub App does not have access to the repository.

---

### 4. `GET /scan/{scan_id}`
Retrieves progress metrics, phase status, and metadata of a specific scan.

* **Path Parameters**:
  - `scan_id` (String, Required): The scan identifier.
* **Response Status**: `200 OK`
* **Response Body**:
  ```json
  {
    "scan_id": "scan-1784201895829",
    "repository": "https://github.com/owner/repo",
    "status": "completed",
    "phase": "ranking",
    "progress": 100,
    "files_scanned": 12,
    "ignored_files": 3,
    "scan_duration_seconds": 12.4
  }
  ```

---

### 5. `GET /bugs/{scan_id}`
Retrieves list of security and reliability bugs discovered during a repository scan.

* **Path Parameters**:
  - `scan_id` (String, Required): The scan identifier.
* **Response Status**: `200 OK`
* **Response Body**:
  ```json
  [
    {
      "id": "bug-0",
      "file": "src/auth.py",
      "line_number": 15,
      "bug_type": "Security",
      "title": "Potential hardcoded secret value",
      "description": "Potential hardcoded credential or secret detected.",
      "severity": "high",
      "confidence": 0.7,
      "impact": "high"
    }
  ]
  ```

---

### 6. `POST /fix/{bug_id}`
Initiates the autonomous fixing agent for a selected bug. Creates a GitHub Issue representing the bug and launches the agent workspace pipeline.

* **Path Parameters**:
  - `bug_id` (String, Required): The ID of the bug from the scan payload.
* **Request Body** (JSON):
  ```json
  {
    "repository_url": "https://github.com/owner/repo",
    "title": "Bug Title",
    "description": "Bug description from finding"
  }
  ```
* **Response Status**: `200 OK`
* **Response Body**:
  ```json
  {
    "status": "issue-created",
    "issue_number": 42
  }
  ```
* **Error Responses**:
  - `400 Bad Request`: Invalid repository URL.

---

### 7. `POST /webhook`
Receives webhook delivery events from GitHub. Validates HMAC signatures before executing.

* **Headers**:
  - `X-Hub-Signature-256` (String, Required): HMAC-SHA256 hash.
  - `X-GitHub-Event` (String, Required): Event context header (e.g., `issues`).
* **Response Status**: `200 OK`
* **Error Responses**:
  - `403 Forbidden`: Signature validation failure.
