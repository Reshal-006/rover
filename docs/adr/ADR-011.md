# ADR-011: Token & Resolution Caching

## Status
Accepted

## Context
1. Creating GitHub App installation access tokens on every API call degrades performance.
2. Executing full LLM fixing cycles on issues that have already been resolved is highly redundant and expensive.

## Decision
1. Implement in-memory token caching. Store generated installation tokens and reuse them until they expire (1 hour).
2. Save successful bug resolutions (patch, test cases, and PR contents) to a local JSON cache file (`scans/resolution_cache.json`). If Rover is re-triggered on the same issue number, it returns the cached result immediately instead of calling the LLM.

## Consequences
Positive:
- Dramatically increases response speeds: token retrieval is instantaneous for cached sessions.
- Bypasses redundant LLM token spend.
- Preserves offline accessibility for previously resolved fixes.

Negative:
- If the remote repository drifts after the resolution is cached, applying the cached patch could cause merge conflicts. In such cases, the cache must be cleared manually.
