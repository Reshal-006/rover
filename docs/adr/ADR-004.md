# ADR-004: GitHub App instead of Personal Access Tokens (PAT)

## Status
Accepted

## Context
Rover needs to interact with GitHub to pull/push repositories, read issues, write comments, and open Pull Requests. Classic configurations use Personal Access Tokens (PATs). However, PATs possess broad user-level access rights, making them a significant security risk if leaked. Furthermore, they are difficult to manage in multi-tenant environments.

## Decision
Transition to GitHub Apps as the primary authentication model, verifying access tokens dynamically using a local private PEM key configuration.

## Consequences
Positive:
- Fine-grained permissions: access can be restricted strictly to Repository Contents, Issues, and Pull Requests.
- Scope-limiting: Access tokens are dynamically requested and limited only to repositories where the user has explicitly installed the App.
- Improved user onboarding: Users can install the App in their repositories via redirection callbacks, rather than manually generating tokens.

Negative:
- Requires managing cryptographic PEM key signing (JWT generation) and installation lookup tokens internally.
