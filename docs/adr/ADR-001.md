# ADR-001: Why Google GenAI (Gemini) function-calling over LangChain

## Status
Draft (to be completed in Week 2)

## Context
Rover needs to orchestrate multiple tool calls in a reasoning loop
where the model decides which function to call and when. The main
choice is between using LangChain agents or the Google GenAI (Gemini)
SDK's native function-calling capability with raw Python.

## Decision
Decision
--------

Use the Google GenAI (Gemini) SDK's function-calling primitives directly
with a raw Python while loop for orchestration. No higher-level agent
framework (e.g., LangChain) is used.

## Consequences
Positive: full control over the loop, easier to debug, zero extra
dependencies, and direct use of the GenAI function-calling primitives
that map well to our tooling model.

Negative: had to write the tool dispatch, message history management,
and retry/error handling manually. LangChain provides these features
out of the box at the cost of added abstraction.

## Alternatives Considered
LangChain: adds abstraction layers that make debugging harder and
encourages different mental models for function-calling flows.

CrewAI: designed for multi-agent systems, overkill for a single-agent
debugging orchestrator.
