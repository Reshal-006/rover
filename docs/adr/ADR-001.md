# ADR-001: Why GPT-4o function calling over LangChain

## Status
Draft (to be completed in Week 2)

## Context
Rover needs to orchestrate multiple tool calls in a reasoning loop
where the model decides which function to call and when. The main
choice is between using LangChain agents or the OpenAI SDK's
native function calling feature with raw Python.

## Decision
Use the OpenAI SDK function calling API directly with a raw Python
while loop for the orchestration. No agent framework is used.

## Consequences
Positive: full control over the loop, easier to debug, zero extra
dependencies, every line is code I understand.
Negative: had to write tool dispatch and message history
management manually. LangChain provides these out of the box.

## Alternatives Considered
LangChain: adds abstraction layers that make debugging harder.
CrewAI: designed for multi-agent systems, overkill for one agent.
