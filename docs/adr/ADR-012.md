# ADR-012: Logging Strategy

## Status
Accepted

## Context
Debugging asynchronous agent operations and background scans is highly difficult without structured execution insights. Furthermore, monitoring LLM cost requires clear tracking of token usage.

## Decision
Configure a centralized logger (`logging.getLogger("rover")`) that:
1. Formats output with timestamps, log levels, and calling modules.
2. Captures and logs prompt and completion token counts on every LLM SDK execution.
3. Outputs step-by-step Git operations, branch conflicts, and retry attempts.
4. Persists execution histories under `logs/` to render runtime summaries in the Streamlit UI.

## Consequences
Positive:
- Complete visibility into agent reasoning, code modifications, and test outcomes.
- Clear reporting of LLM token spend helps control development and deployment costs.
- Integrates seamlessly with terminal outputs and dashboard log viewers.

Negative:
- Large logs consume disk space over time. Log files should eventually implement rotation rules.
