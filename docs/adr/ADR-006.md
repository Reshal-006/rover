# ADR-006: FastAPI Backend API

## Status
Accepted

## Context
Rover needs to receive webhook events from GitHub in real-time, execute long-running repository scan and fix pipelines asynchronously, and expose JSON endpoints for the dashboard and browser extensions.

## Decision
Use FastAPI as the backend framework, running background tasks via FastAPI's native `BackgroundTasks` library.

## Consequences
Positive:
- Asynchronous endpoint handling ensures quick responses to GitHub webhooks, keeping operations within limits.
- Built-in JSON serialization and Pydantic validation simplify API contracts.
- Auto-generated interactive Swagger UI API docs (`/docs`).

Negative:
- Running long background processes inside the same service thread can compete for CPU resources. For heavy scale, background operations should migrate to Celery or Redis Queues.
