# ADR-008: Dynamic Branch Naming & Conflict Resolution

## Status
Accepted

## Context
When Rover pushes changes to remote repositories, reusing static branch names (such as `rover/fix-issue-6`) frequently causes `non-fast-forward` git push rejects because remote states drift or branches already exist. 

## Decision
1. Generate unique feature branch names with datetime and short UUID suffixes: `rover/fix-issue-{number}-{timestamp}-{short_uuid}`.
2. Query the GitHub API to ensure the branch does not exist on remote origin before creating it.
3. If the push fails because of unexpected remote branch conflicts, automatically rename the local branch and retry the push-and-PR flow up to 3 times without manual intervention.

## Consequences
Positive:
- Eliminates `non-fast-forward` and push collision errors entirely.
- Preserves clean Git histories without merging previous AI branches or performing force pulls.
- Fully automated recovery keeps the execution pipeline robust.

Negative:
- Leaves multiple unique remote branches on the target repository. Scans should eventually include post-merge remote branch cleanup.
