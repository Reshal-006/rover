# ADR-007: Local Repository Cloning

## Status
Accepted

## Context
Rover needs to parse syntax trees, inspect imports, run unit tests, and compile file edits. There are two primary approaches:
1. Query GitHub APIs for file contents dynamically (and perform all reasoning in-memory).
2. Clone repositories into local workspaces and perform operations locally.

## Decision
Clone repositories into local sub-directories (`workspace/`) using Git commands. 

## Consequences
Positive:
- Full accessibility to all codebase contents without hitting GitHub API rate limits.
- Supports running real local tests (`pytest`) directly on the actual codebase structure.
- Facilitates the use of standard terminal-based static analysis and AST parsers.

Negative:
- Cloned workspaces consume disk space and must be cleaned up or reset regularly.
- Running untrusted code locally introduces safety/security concerns if the repository contains malicious test setups.
