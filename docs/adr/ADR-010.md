# ADR-010: Repository Indexing before LLM

## Status
Accepted

## Context
Iterative "Think -> Search -> Think" agent loops make multiple consecutive LLM calls to locate relevant files and symbols. This process is extremely slow and exhausts token quotas rapidly.

## Decision
Implement a local Abstract Syntax Tree (AST) indexer (`src/indexer.py`) that extracts symbols (classes, functions, imports) on target clones prior to invoking the LLM. 

## Consequences
Positive:
- Dramatically reduces LLM token consumption and run duration.
- Enables single-call structured response prompt modeling ("Analyze -> Gather Context -> Solve") by bundling all symbol mappings directly into the context.
- Eliminates the need for the LLM to write search heuristics.

Negative:
- Requires custom AST parsers. Python parsing is robust, but multi-language support requires parsing configurations (like Tree-sitter).
- Increases initial execution time for large codebases to build the index.
*Note: This index is cached to disk (`.rover_index.json`) and warmed up on clone.*
